# 中古せどりチェックリスト 公開用

公開ページ:

https://solu-star.github.io/resale-checklist/

GitHubリポジトリ:

https://github.com/Solu-star/resale-checklist

## 使い方

- スマホでは公開URLをブックマーク、またはホーム画面に追加して使う。
- 内容を更新したら `index.html` を編集してGitHubへ反映する。
- 画面右上の「更新」ボタンで、スマホ側のキャッシュを避けて読み直せる。

## 機能

- **巡回カレンダー**: 月表示のカレンダーで巡回予定と実績を一覧できる。日付をタップすると、その日の予定の追加・削除、回った店舗の確認ができる。1日に回った店舗数は「3店」のように日付の下に表示される。
- **巡回予定**: カレンダーの日付をタップして店舗名を入力すると予定として保存される(その端末だけに保存)。
- **巡回履歴**: 日付・店舗・コーナー・メーカー・メモを記録できる。「昨日」「今日」ボタンで日付をすぐ入れられる。カレンダーの「この日付で巡回を記録する」からも入力できる。
- **チェックの保存**: チェックはその端末の中に自動保存され、ページを閉じても消えない(端末ごとに別々)。
- **チェック全消し**: 店舗を回り終えたら「チェック全消し」ボタンでまとめてリセットできる。
- **全部閉じる/開く**: 画面上部のボタンで、すべてのコーナーをまとめて閉じたり開いたりできる。
- **コーナーへジャンプ**: 検索欄の下のボタンを押すと、そのコーナーまで一気に移動する。
- **上へ戻る**: 下にスクロールすると右下に「↑ 上へ」ボタンが出て、ワンタップで先頭に戻れる。
- **検索**: ブランド単位で絞り込み。ひらがな/カタカナ、全角/半角、大文字/小文字の違いは気にしなくてよい。
- **メモ**: 各コーナーの下のメモ欄に書くと自動保存される(その端末だけに保存)。
- **件数表示**: コーナー名の右に「チェック済み/全体」の数が出る。
- **開閉の記憶**: コーナーの開く/閉じるの状態を覚えておく。
- **ダークモード**: スマホの設定がダークモードなら自動で暗い配色になる。

## 他のPCで作業を続ける方法

1. GitHubにログインできるPCでターミナルを開く。
2. 作業したいフォルダへ移動する。
3. 次のコマンドでリポジトリを取得する。

```bash
git clone https://github.com/Solu-star/resale-checklist.git
cd resale-checklist
```

4. `index.html` を編集する。
5. ローカルで確認する。

```bash
python3 -m http.server 8765
```

ブラウザで `http://127.0.0.1:8765/` を開く。

6. 問題なければGitHubへ反映する。

```bash
git status
git add index.html README.md
git commit -m "Update checklist"
git push
```

7. 数十秒から数分後に公開ページへ反映される。

## 更新するときの注意

- `index.html` の `dataUpdatedAt` を更新日時に合わせて変更する。
- スマホで古い表示が残る場合は、画面上部の「更新」ボタンを押す。
- 変更前に `git pull` してから作業すると、他PCでの変更とぶつかりにくい。

```bash
git pull
```

## 注意

- 公開URLを知っている人は閲覧できるため、仕入れノウハウを完全に非公開にしたい場合は、パスワード付きの公開先を使う。
